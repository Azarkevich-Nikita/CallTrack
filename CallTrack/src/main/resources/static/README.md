# CallTrack - Система учета звонков

## Структура проекта

### HTML файлы
- `index.html` - главная страница
- `tariffs.html` - страница тарифов (реструктурирована)
- `registration_payments.html` - регистрация платежей
- `registration_calls.html` - регистрация звонков
- `reports_payments.html` - отчеты по платежам
- `reports_calls.html` - отчеты по звонкам
- `notifications_debt.html` - уведомления о долгах

### Стили (styles/)
- `common.css` - общие стили для всех страниц
- `tariffs.css` - специфичные стили для страницы тарифов
- `README.md` - документация по стилям

### Скрипты (scripts/)
- `tariffManager.js` - логика управления тарифами
- `clientSearch.js` - поиск клиентов
- `ui.js` - общие UI функции
- `README.md` - документация по скриптам

### Ресурсы (assets/)
- `images/` - изображения и иконки

## Реструктуризация tariffs.html

### Что было сделано:
1. **Разделение CSS** - вынесены стили в отдельные файлы
2. **Разделение JavaScript** - логика вынесена в отдельный модуль
3. **Упрощение HTML** - убраны встроенные стили и скрипты
4. **Модульность** - создана структура для легкого сопровождения

### Преимущества новой структуры:
- ✅ Легче поддерживать и изменять код
- ✅ Переиспользование стилей и скриптов
- ✅ Лучшая организация файлов
- ✅ Упрощенная отладка
- ✅ Модульность компонентов

### Подключение файлов:
```html
<!-- Стили -->
<link rel="stylesheet" href="styles/common.css">
<link rel="stylesheet" href="styles/tariffs.css">

<!-- Скрипты -->
<script src="scripts/tariffManager.js"></script>
```

## Разработка

### Добавление новых страниц:
1. Создать HTML файл
2. Подключить `common.css` для базовых стилей
3. Создать специфичный CSS файл при необходимости
4. Создать JavaScript модуль для логики

### Модификация существующих страниц:
1. Изменить соответствующий CSS файл
2. Обновить JavaScript модуль
3. HTML остается чистым и простым

## Технологии
- HTML5
- CSS3 (с переменными)
- Vanilla JavaScript (ES6+)
- Responsive Design
